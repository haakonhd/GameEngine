GameEngine

Setting up Visual studio

To use the game engine you have to download Visual studio 2019. 2017 should work as well, but it has not been tested. Use the Visual Studio Installer to modify your Visual studio installation to include Universal Windows Development under the "workloads" tab. You may need to turn on developer mode in windows to run UWP applications. If you are using Mac, you can follow this tutorial.

Running the project

At the root of the project folder structure, there is a file called GameEngine.snl. Double-clicking this should open project with Visual Studio. To run the game, select "debug" or "release", and "x64" next to the play button on the top of the page. The play button will launch the game.

Selecting games

When the game starts, it starts by running the MainPage.xaml.cs file. From here you can select which game implementation to run in the Navigate function.In this project, we have two implementations, the Empty Project, and Pokemon. These games can be found in the "implementation" folder.

Creating a game

When creating a game, it is recommended to use the empty project, and change the sample code to fit your game, and expand from there. This project contains detailed descriptions in the code on what what steps are needed to create a basic game, and how different functionalities of the engine work. The game is set up in the GameInitializer.xaml.cs file.
